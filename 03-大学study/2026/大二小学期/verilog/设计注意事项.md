
# 📝 FPGA/IC 设计注意事项笔记

## 📌 01. 禁用门控时钟，改用时钟使能端

### 🔍 核心原则

> **永远不要对时钟信号（Clock）进行任何逻辑操作（如与、或、非等门电路加工）后，再作为触发器的时钟输入！**
> 
> 必须保持全局时钟的绝对纯净。如果需要控制触发器是否工作，一律通过控制触发器的 **使能端（ENA / Clock Enable）** 实现。

### ❌ 门控时钟（Gated Clock）—— 错误示范 ☹️

#### 1. 电路物理本质

控制信号（`input`）和主时钟（`clk`）一起接入了组合逻辑门（例如一个与门），“加工”出了一条新的、不稳定的时钟线连接到触发器的时钟引脚（▷）。

#### 2. 致命隐患：毛刺（Glitches）与误动作

- **原因**：组合逻辑存在物理走线延时。当控制信号抖动或与时钟边缘重合时，由于延时差，组合逻辑输出会产生极窄的**有害毛刺**。
    
- **后果**：触发器会将毛刺误判为真实的“时钟上升沿”，导致电路在错误的时间点抓取数据，系统直接**跑飞 / 逻辑崩溃**。
    

#### 3. 错误 Verilog 代码（无意间引入门控时钟）

Verilog

```
// ⚠️ 严重警告：敏感列表混入控制信号，极易被综合为门控时钟
always @(posedge clk or posedge input_signal) begin
    if (input_signal) begin
        q <= d;
    end
end
```

### 时钟使能端（Clock Enable）—— 规范写法 😃

#### 1. 电路物理本质

主时钟（`clk`）走 FPGA 内部专用的**全局时钟网络（Clock Tree）**，干净、直接地连到触发器的时钟引脚（▷）。而控制信号（`input`）经过组合逻辑后，去连触发器的**使能引脚（ENA）**。

#### 2. 为什么安全？

- 主时钟没有毛刺，逻辑极其稳定。
    
- 每当 `clk` 上升沿到来时，触发器先检查 `ENA` 的状态（1 则更新数据，0 则保持原值）。
    
- 即使 `ENA` 线上有毛刺，只要毛刺在时钟沿到来前稳定下来（满足建立/保持时间），电路就**绝对不会发生误动作**。
    

#### 3. 规范 Verilog 代码（工业级标准）

Verilog

```
//  标准写法：敏感列表只有时钟和复位，控制信号写在内部
always @(posedge clk or negedge reset) begin
    if (!reset) begin
        q <= 1'b0; // 异步复位
    end
    else if (en_signal) begin
        q <= d;    // 当使能信号(PPT中的组合逻辑输出)有效时，时钟沿到来更新数据
    end
    else begin
        q <= q;    // 使能无效时，数据安全保持
    end
end
```

### 💡 考试/面试大白话总结

- **时序逻辑的敏感列表（`always @`）里永远只允许出现纯净的时钟和复位。**
    
- **控制条件（如开关、使能、计数到达）永远只能写在 `always` 块内部的 `if-else` 条件句中！**

## 📌 02. 慎重对计数器进行译码（组合逻辑毛刺与编码选择）

### 🔍 核心原则

> **对普通二进制计数器的输出进行组合逻辑译码（如解码、判断特定数值）时，输出信号极易因为“竞争冒险”产生有害毛刺！**
> 
> 如果后级是纯同步时序电路，可以不予理会；但如果该译码输出要用作时钟、复位、或异步控制信号，则必须通过改变**计数器编码方式**（如格雷码、独热码）或**寄存器打拍**来消除毛刺。

### ❌ 问题的根源：二进制计数器的“竞争冒险”

#### 1. 现象与成因

普通的二进制计数器在状态跳转时，可能存在**多个位同时改变**的情况。

- 举例：计数器从 `3`（`4'b0011`）跳转到 `4`（`4'b0100`）。
    
- 在物理芯片内部，由于走线长度和逻辑门延迟的不同，低三位的变化不可能绝对同步。可能会短暂经历一个错误中间态（如 `4'b0000` 或 `4'b0111`）。
    
- 此时，后级的**组合逻辑译码器**会瞬间捕捉到这个错误的中间状态，并在输出端（`out`）吐出一个极窄的脉冲——这就是**毛刺（Glitch）**。
    

#### 2. 什么时候可以“不予理会”？

- **条件**：如果译码器的输出 `out` 后面接的是一个标准的**同步时序电路**（即 `out` 只连到了下一个触发器的 **D 数据输入端**）。
    
- **原因**：触发器只在 `clk` 的上升沿那一瞬间采样。只要毛刺发生在线路传输过程中（时钟沿之间），并且在下一个时钟沿到来前数据已经恢复稳定（满足建立时间），这个毛刺就不会带来任何危害。
    

### 解决方案：高级编码与避坑设计 😃

当毛刺对电路有危害（例如作为异步控制引脚）时，可以采用以下两种计数器编码方式替代传统的二进制码：

#### 1. 格雷码（Gray Code）—— 适合 PLD/特定跨时钟域

- **特点**：任意两个相邻的数值之间，**只有一位（1 bit）发生改变**。
    
- **优势**：从源头上消除了一次改变多位带来的中间错误状态，译码输出绝对不会产生竞争冒险导致的毛刺。
    

#### 2. 独热码（One-hot Code）—— FPGA 器件的首选

- **特点**：$N$ 个状态由 $N$ 个触发器表示，任何时候有且仅有一位是 `1`（例如：`0001` $\rightarrow$ `0010` $\rightarrow$ `0100` $\rightarrow$ `1000`）。
    
- **优势**：FPGA 内部拥有极其丰富的触发器资源（Flip-Flops），但组合逻辑资源（LUT）相对紧缺。独热码由于状态极度稀疏，其对应的译码器（判断语句）电路非常简单，几乎不需要复杂的组合逻辑，能够有效提高电路运行的最高时钟频率（$F_{max}$）。
    

#### 3. 终极万能药：寄存器打拍（时序输出）

除了换编码，最常用的通用规范是在组合逻辑译码输出后，**加一级触发器强行锁存（打一拍）**，把组合逻辑毛刺变成干净的时序输出：

Verilog

```
reg out_reg;
// 将具有毛刺的组合逻辑输出 out 接入触发器，变成绝对干净的同步信号 out_reg
always @(posedge clk or negedge reset) begin
    if (!reset)
        out_reg <= 1'b0;
    else
        out_reg <= out; // 打一拍，消除毛刺
end
```

### 💡 考试/面试大白话总结

- 普通二进制计数器变化时，由于物理延迟，多位跳转会产生**组合逻辑毛刺**。
    
- **后级是 D 触发器数据端** $\rightarrow$ 安全，不用理会。
    
- **后级是时钟/复位/异步使能** $\rightarrow$ 危险！必须换成**独热码/格雷码计数器**，或者在译码输出后**用触发器打一拍**过滤毛刺！

这个知识点讲的是数电/FPGA 设计中最经典、也最容易让人抓狂的语法硬伤——**多驱动问题（Multi-driver / Wire tri-stated conflict）**。还记得我们在最开始修改你那段数码管代码时提到的第一条致命硬伤吗？本质上就是这个问题。

## 📌 03. 避免多驱动问题（Multi-driver Conflict）

### 🔍 核心原则

> **在可综合的 Verilog 设计中，任何一个信号（线网型 `wire` 或寄存器型 `reg`）有且只能有一个硬件驱动源！**
> 
> 严禁在两个或多个不同的 `always` 块（或 `assign` 语句）中对同一个信号进行赋值。

### ❌ 问题的物理本质：短路与电压冲突 ☹️

#### 1. 硬件电路的真相

在真实的芯片物理结构中，给信号赋值（如 `out = 1` 或 `out = 0`）本质上是通过互补的晶体管（PMOS/NMOS）将导线连接到**电源（VCC）**或**地（GND）**。

- 如果你在**电路块 1** 里让 `out = 1`（导线接高电平）；
    
- 同时在**电路块 2** 里让 `out = 0`（导线接低电平）；
    
- 这两条导线一旦接在同一个物理节点上，就会发生**电源和地直接短路**！物理上会产生很大的电流导致芯片发热甚至烧毁；在逻辑上，综合工具无法判定这个点到底是高还是低，只能将其判定为未知态（X）并直接报错。
    

#### 2. 常见错误 Verilog 代码示例

Verilog

```
// ❌ 错误示范：在两个独立的 always 块中驱动同一个变量 counter
always @(posedge clk) begin
    if (cond1) counter <= counter + 1'b1;
end

always @(posedge clk) begin
    if (cond2) counter <= counter - 1'b1; // 综合报错：Signal 'counter' has multiple drivers!
end
```

### 正确解决方案：多路合一（Multiplexing） 😃

如果你希望在不同的条件（比如不同的控制信号）下改变同一个变量，**必须将这些所有的条件合并到同一个 `always` 块（或同一条 `assign` 语句）中**，通过 `if-else` 或 `case` 转换为硬件上的**数据选择器（MUX）**。

#### 1. 正确的 Verilog 代码（时序逻辑合并）

Verilog

```
//  标准写法：所有针对 counter 的改变行为，全部塞进同一个控制核心内
always @(posedge clk or negedge reset) begin
    if (!reset) begin
        counter <= 32'd0;
    end
    else if (cond1) begin
        counter <= counter + 1'b1; // 条件1 触发
    end
    else if (cond2) begin
        counter <= counter - 1'b1; // 条件2 触发
    end
    else begin
        counter <= counter;        // 默认保持
    end
end
```

#### 2. 补充：组合逻辑拆分赋值的特例（还记得你最初的代码吗？）

在同一个 `always @*` 块内，虽然允许对同一个变量的不同位（bit slice）或者先后赋值，但极易因为漏掉条件产生 Latch。最优的标准依然是在**同一个块的开头赋全局默认值**，或者利用**花括号拼接 `{}`** 一口气完成赋值。

- **错误写法**：在块 A 里写 `sseg[7:1] = ...`，在块 B 里写 `sseg[0] = ...`（多驱动报错）。
    
- **正确写法**：在同一个 `always @*` 里写 `sseg = {7'bxxxxxxx, dp};`（唯一驱动，安全通过）。
    

### 💡 考试/面试大白话总结

- 一个信号在现实中只有一根导线，**不能有两个“高管”同时指挥它**。
    
- 只要看到报错提示 **"X has multiple drivers"**，不用怀疑，立刻去代码里全文搜索这个变量，把它从别的 `always` 块或 `assign` 里删掉，**合并成一个块**！

## 📌 04. 多时钟的同步化（跨时钟域 CDC 基础）

### 🔍 核心原则

> **绝对禁止将一个时钟域（`clk1`）下的数据，直接经过组合逻辑后送入另一个没有任何相位关系的异步时钟域（`clk2`）中进行采样！**
> 
> 异步时钟域之间传输数据极易导致**亚稳态（Metastasis）**蔓延。必须在目标时钟域（`clk2`）的入口处使用触发器进行**同步化（Synchronization）**处理。

### ❌ 异步跨时钟直接采样的致命灾难 ☹️

#### 1. 电路物理本质（哭脸电路）

前级触发器由 `clk1` 驱动，产生的信号经过一段组合逻辑（带有线网延迟），直接连到了由 `clk2` 驱动的后级触发器的 **D 数据输入端**。

#### 2. 致命隐患：亚稳态（Metastability）

- **成因**：由于 `clk1` 和 `clk2` 是异步的（没有固定的相位关系，各自跑各自的），数据在 `clk1` 下发生改变的瞬间，极有可能正好撞上 `clk2` 的上升沿。
    
- **物理后果**：此时，后级触发器的 D 端数据由于没有保持稳定，违反了**建立时间（Setup Time）**或**保持时间（Hold time）**。触发器内部的物理晶体管会陷入“既不是高电平，也不是低电平”的中间震荡状态。
    
- **灾难蔓延**：这个不确定的电平随后会穿过组合逻辑，导致后级整个数字系统的所有触发器全部状态混乱，芯片彻底锁死崩溃。
    

### 正确解决方案：触发器同步化（两级同步器） 😃

为了防止亚稳态在目标时钟域 `clk2` 内部蔓延，必须使用目标时钟域的触发器对异步信号进行**同步化（打拍）**。

#### 1. 改进后的电路本质（笑脸电路）

- 异步信号从 `clk1` 过来到 `clk2` 域后，**直接接入 `clk2` 驱动的触发器 D 端（中继站）**。
    
- 即使第一级触发器因为跨时钟采样产生了亚稳态，它也有整整一个时钟周期（`clk2` 的周期）的时间去让其内部电压恢复稳定（沉降）。
    
- 经过一到两级触发器的锁存（即工业界著名的 **两级同步器 / 两级打拍**），传给后级组合逻辑的数据就是 100% 干净、稳定的单轨电平。
    

#### 2. 规范 Verilog 代码（异步单比特信号同步化）

Verilog

```
reg sync_reg1; // 第一级同步寄存器（用于将亚稳态锁死在这一级）
reg sync_reg2; // 第二级同步寄存器（输出给后续组合逻辑使用，绝对安全）

// 所有的同步操作，必须在目标时钟域 clk2 内进行
always @(posedge clk2 or negedge reset) begin
    if (!reset) begin
        sync_reg1 <= 1'b0;
        sync_reg2 <= 1'b0;
    end
    else begin
        sync_reg1 <= data_from_clk1; // 第一拍：允许发生亚稳态，但将其定格在这里
        sync_reg2 <= sync_reg1;      // 第二拍：输出干净、不再抖动的稳定同步信号
    end
end

// 后续的组合逻辑，一律只能使用 sync_reg2 作为输入！
assign out = sync_reg2 & other_signal;
```

### 💡 考试/面试大白话总结

- 两个不同的时钟就像两个说不同语言的人，**不能面对面直接传话**，否则会因为“听不清（违反时序）”导致脑子短路（**亚稳态**）。
    
- **解法**：在听话的那方（`clk2`）门口安排两个哨兵（**两级同步触发器**），哪怕第一个哨兵听懵了（产生亚稳态），等传到第二个哨兵手里时，信息也一定是清晰、确定的了。这个过程就叫**跨时钟域打拍同步**！
