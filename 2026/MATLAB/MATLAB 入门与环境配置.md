# 🛠️ MATLAB 入门与环境配置

## 1. 界面布局与核心窗口

MATLAB 的默认界面（Desktop）由以下核心组件构成，建议通过 `布局 (Layout)` 锁定为“默认”以方便操作。

- **当前文件夹 (Current Folder)**：MATLAB 运行脚本和读取数据的默认查找位置。
    
- **命令行窗口 (Command Window)**：直接输入指令并立即执行（交互式环境）。
    
- **工作区 (Workspace)**：记录当前内存中存储的所有变量名、值及类型。
    
- **命令历史记录 (Command History)**：追踪以往输入过的所有代码。

## 2. 路径设置 (Path Management)

> [!IMPORTANT] 为什么路径很重要？
> 
> 当你在命令行输入 `my_function` 时，MATLAB 会按以下顺序搜索：
> 
> 1. 内存中的变量。
>     
> 2. 当前文件夹下的 M 文件。
>     
> 3. **搜索路径 (Search Path)** 列表中的文件夹。
>     
>     如果你的脚本不在上述范围内，MATLAB 会报错：“未找到函数或变量”。
>     

### 设置方法

1. **命令行修改**：
    
    - `addpath('C:\MyProject')`：临时将文件夹加入路径。
        
    - `savepath`：将当前路径修改永久保存。
        
2. **图形界面**：点击主页 (Home) 选项卡 -> **设置路径 (Set Path)**。

## 3. 工作区管理 (Workspace Control)

在进行 [[MATLAB 数值计算]] 或信号处理实验前，保持工作区干净是避免变量污染的关键。

### 核心管理指令

|**指令**|**功能**|**备注**|
|---|---|---|
|`who`|列出当前工作区所有变量名||
|`whos`|查看变量的详细信息（维度、字节、类）|检查矩阵维度非常有用|
|`clear`|清除工作区**所有**变量|脚本开头必备|
|`clear var1`|仅清除变量 `var1`||
|`clc`|清除命令行窗口的所有文本|不影响内存变量|
|`save data.mat`|将工作区变量保存到磁盘|生成 `.mat` 文件|
|`load data.mat`|将保存的变量读取回内存||

## 4. 环境配置与效率技巧

### 4.1 常用快捷键

- `Ctrl + R` / `Ctrl + T`：注释 / 取消注释代码块。
    
- `F5`：保存并运行当前脚本。
    
- `Tab`：自动补全函数名或变量名。
    
- `↑` (方向键)：快速调出上一条执行过的命令。
    

### 4.2 脚本开头“三件套”

在编写任何关于 [[信号与系统的 MATLAB 实现]] 的脚本时，建议养成以下习惯：

Matlab

```
%% 初始化环境
clear;      % 清除变量
clc;        % 清屏
close all;  % 关闭所有打开的图像窗口（Figure）
```
